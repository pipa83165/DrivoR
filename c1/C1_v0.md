# C1_v0 改动统计与使用说明

生成时间：2026-07-04

## 实现范围

本版按照 `overview.md`、`C1.md`、`C1_code_changes.md` 对齐新版 C1：冻结 VGGT-Omega 1B 几何分支，训练阶段从离线缓存读取几何 token，在 DrivoR decoder memory 处拼接，不改 DrivoR 主干 ViT、不改 decoder 结构、不改损失。C1 是实验路径，不是新模型名称；默认仍关闭，只有显式设置 `agent.config.vggt_geometry.enabled=true` 时启用。

新版实现保持轻量：VGGT geometry 分支逻辑集中在 `navsim/agents/drivoR/vggt_geometry.py`，主模型只负责把几何 token 追加到 decoder memory。

## 关键改动

1. `VggtGeometryProjector` 改成轻量直观结构：`LayerNorm -> Linear(2048,256) -> +branch/camera embed -> LayerNorm -> zero LayerScale gate`。初始化时输出严格为 0，避免随机几何 token 一开始干扰 decoder。
2. `drop` 对照不再置零，而是在评测时直接不拼接几何 token，decoder memory 回到原始长度 64，避免 zero key 分走 attention。
3. `shuffle/noise` 下沉到 `VggtGeometryTokenProvider`，从 split 缓存池随机配对或用缓存全体均值方差造噪声；不依赖 batch 内打乱，所以 batch=1 也能跑。
4. 缓存/训练强制校验 metadata 指纹。checkpoint、VGGT dim、相机顺序、官方预处理参数、`load_fn.py` 文件 hash、是否 joint forward、是否使用 camera token 都会参与严格校验。
5. 在线预处理和离线缓存都复用 VGGT-Omega 官方 `load_fn` 规则：默认 `preprocess_mode=balanced`，输入保持 `[0,1]` RGB，ImageNet mean/std 归一化只在 VGGT Aggregator 内部做一次。
6. VGGT-Omega 教师分支保持冻结：`eval()`、`torch.no_grad()`、`requires_grad_(False)`；offline cache 天然无梯度。可训练的是 VGGT geometry adapter 的 `geo_proj`、branch embedding、camera embedding。

## 默认配置

配置位置：`navsim/planning/script/config/common/agent/drivoR.yaml`

```yaml
agent:
  config:
    vggt_geometry:
      enabled: false
      mode: normal            # normal | shuffle | noise | drop
      source: cache           # cache for training; online for eval/latency
      cache_dir: null
      checkpoint_path: weights/vggt_omega_1b_512.pt
      vggt_dim: 2048          # frame-attn(1024) || global-attn(1024)
      num_registers: 16
      use_camera_token: false
      joint_forward: true
      preprocess_mode: balanced
      image_resolution: 512
      shuffle_seed: 20260704
      force_ignore_fingerprint: false
```

C1 训练/验证配置现在写在 `c1/train_c1.sh` 和 `c1/validate_c1.sh` 中。脚本直接调用默认训练配置，并像 README 一样用命令行 override 写清 C1 参数。

脚本内关键覆盖：

- `agent.config.vggt_geometry.enabled=true`
- `agent.config.vggt_geometry.source=cache`
- `agent.config.vggt_geometry.mode=$MODE`
- `agent.config.vggt_geometry.cache_dir=$VGGT_GEOMETRY_CACHE_DIR`
- `agent.lr_args.base_lr=$BASE_LR`，默认 `0.0002`
- `dataloader.params.batch_size=$BATCH_SIZE`，默认 `16`
- `trainer.params.max_epochs=$MAX_EPOCHS`，默认 `10`
- `output_dir=$REPO_ROOT/c1/${EXPERIMENT_NAME}/${experiment_uid}`

## 使用方法

先进入项目并设置环境变量：

```bash
cd /high_perf_store3/world-model/weixiaobao/yzj/DrivoR

export NUPLAN_MAP_VERSION="nuplan-maps-v1.0"
export NUPLAN_MAPS_ROOT="./dataset/maps"
export NAVSIM_EXP_ROOT="./exp"
export NAVSIM_DEVKIT_ROOT="./"
export OPENSCENE_DATA_ROOT="./dataset"
export HYDRA_FULL_ERROR=1
export SUBSCORE_PATH=$NAVSIM_EXP_ROOT
```

### 1. 轻量验收

```bash
python3 -m py_compile \
  navsim/agents/drivoR/vggt_geometry.py \
  navsim/agents/drivoR/drivor_model.py \
  navsim/agents/drivoR/drivor_features.py \
  navsim/planning/training/dataset.py \
  navsim/planning/script/run_training_full.py

python3 scripts/vggt_geometry_acceptance_checks.py
```

正确结果：最后出现

```text
PASS: lightweight VGGT geometry acceptance checks passed
```

### 2. 检查在线预处理

```bash
python3 c1/check_vggt_geometry_preprocess.py --image /path/to/one_camera_image.jpg
```

正确结果应满足：

- `dtype=torch.float32`
- `min >= 0` 且 `max <= 1`
- height/width 是 16 的倍数
- `source check: OK`
- `internal normalize: OK`
- 最后一行是 `PASS`

这里的 shape 不再固定为旧版 `(3,336,448)`；新版默认 `balanced` 会按官方规则得到 patch16 对齐尺寸。

### 3. 生成 C1 token 缓存

建议不要覆盖旧缓存，先写到新目录：

```bash
python3 -m navsim.agents.drivoR.scripts.cache_vggt_geometry_tokens \
  --checkpoint weights/vggt_omega_1b_512.pt \
  --output-dir c1/vggt_geometry_tokens_new \
  --split trainval \
  --preprocess-mode balanced \
  --forward-mode joint \
  --skip-existing
```

脚本输出内容：

- 每个 sample token 一个 fp16 `.pt`，默认形状 `[4,16,2048]`；
- `metadata.json`：强制指纹信息；
- `token_index.json`：token 到缓存文件的映射；
- `noise_stats.pt`：C1-noise 对照使用的 per-dim 均值/方差。

如果后续训练报 fingerprint mismatch，说明缓存不是当前配置生成的。正确处理是重新生成缓存，除非只是临时排查，否则不要开 `force_ignore_fingerprint=true`。

### 4. 训练 C1 主实验

推荐直接使用 `c1/` 目录下的训练脚本；C1 训练配置已经写在脚本里：

```bash
bash c1/train_c1.sh
```

等价于 `MODE=normal`。如果缓存、batch、epoch 或学习率要改，用环境变量覆盖：

```bash
VGGT_GEOMETRY_CACHE_DIR=c1/vggt_geometry_tokens_new \
BATCH_SIZE=16 \
MAX_EPOCHS=10 \
BASE_LR=0.0002 \
bash c1/train_c1.sh
```

### 5. 训练内部对照

同一个脚本通过 `MODE` 切换对照：

```bash
MODE=shuffle bash c1/train_c1.sh
MODE=noise bash c1/train_c1.sh
MODE=drop bash c1/train_c1.sh
```

也可以覆盖实验名和 seed：

```bash
MODE=shuffle EXPERIMENT_NAME=C1_shuffle_seed2 SEED=2 bash c1/train_c1.sh
```

### 6. 验证 C1 checkpoint

验证脚本放在 `c1/validate_c1.sh`，必须指定 checkpoint：

```bash
CKPT_PATH=/path/to/checkpoint.ckpt bash c1/validate_c1.sh
```

验证对照同样用 `MODE` 切换：

```bash
MODE=shuffle CKPT_PATH=/path/to/checkpoint.ckpt bash c1/validate_c1.sh
MODE=noise CKPT_PATH=/path/to/checkpoint.ckpt bash c1/validate_c1.sh
MODE=drop CKPT_PATH=/path/to/checkpoint.ckpt bash c1/validate_c1.sh
```

解释：

- `normal`：读取本样本真实 VGGT 几何 token。
- `shuffle`：读取另一个随机样本的真实 VGGT token，检验是否只是容量效应。
- `noise`：用缓存全体均值/方差生成噪声 token，检验 decoder 是否只是利用额外 token 位置。
- `drop`：训练带几何 token，评测时物理移除几何 token，观察模型依赖度。


## 注意事项

- C1 的融合点只在 decoder memory，不能把 VGGT token 喂回 DrivoR ViT 主干，否则实验语义会变。
- VGGT-Omega 分支是冻结 teacher；VGGT geometry adapter 参数是可训练学生侧参数。
- 如果要做在线延迟评测，将配置改为 `agent.config.vggt_geometry.source=online`，此时会现场跑冻结 VGGT-Omega，推理成本会计入延迟。
